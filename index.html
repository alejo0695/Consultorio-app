Instrucciones de despliegue con Google Sheets y Apps Script

  1. Crear el Google Sheet
    • Crea una hoja de cálculo en Google Drive llamada por ejemplo ConsultorioDB.
    • Crea tres hojas con estos nombres exactos: Usuarios, Pacientes, Pagos.
    • En Usuarios pon la fila 1 como encabezados: usuario, password, rol.
    • Agrega algunos usuarios de prueba, por ejemplo: admin | 1234 | ADMIN y secretaria | 1234 | SECRETARIA.
    • En Pacientes usa encabezados: timestamp, nombre, documento, telefono, empresa, terapeuta.
    • En Pagos usa encabezados: timestamp, paciente, fecha, monto, metodo, terapeuta, empresa.
  2. Crear el proyecto de Google Apps Script
    • Desde el Sheet, ve a Extensiones → Apps Script.
    • Elimina cualquier código existente en Code.gs.
    • Pega el contenido del archivo Code.gs que se incluye en este proyecto (ver más abajo).
    • En la parte superior, haz clic en Guardar y ponle un nombre al proyecto.
  3. Configurar la URL del Web App
    • En Apps Script, ve a Implementar → Nueva implementación.
    • Tipo de implementación: Aplicación web.
    • Ejecutar como: Tú.
    • Quién tiene acceso: Cualquiera con el enlace (o solo tu dominio, según necesidad).
    • Haz clic en Implementar y copia la URL del Web App, algo como https://script.google.com/macros/s/XYZ/exec.
  4. Conectar el frontend con el backend
    • En este proyecto, abre el archivo script.js.
    • Busca la constante API_BASE_URL y reemplaza su valor por la URL de tu Web App.
    • Sube los archivos index.html, style.css y script.js a cualquier hosting estático (por ejemplo GitHub Pages, Netlify o incluso abre el index.html localmente para pruebas).
  5. Probar el sistema
    • Abre el index.html en el navegador.
    • Inicia sesión con uno de los usuarios creados en la hoja Usuarios.
    • Prueba el registro de pacientes y pagos.
    • Si entras como ADMIN verás el dashboard y el módulo de cobros; como SECRETARIA solo verás las opciones de registro.

Consejo: si cambias el nombre de las hojas o agregas más columnas, recuerda actualizar tanto el código de Apps Script como las transformaciones de datos en script.js.
